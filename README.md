# 영어 스피킹 연습 앱 (English Speaking Practice App)

## 📖 프로젝트 소개

이 앱은 **영어 스피킹을 체계적으로 연습하고 싶은 학습자를 위한 AI 기반 대화형 연습 플랫폼**이다. 한국어-영어 문장 쌍을 등록하는 것부터 시작해서, 랜덤으로 한국어 문장을 읽어주면 사용자가 영어로 답하고, 실시간으로 피드백을 받는 전 과정을 제공한다.

### 핵심 기능

1. **문장 등록하기**
   - 한국어-영어 문장 쌍을 직접 입력하거나 이미지로 업로드한다
   - OCR이나 AI로 사진 속 문장을 자동으로 추출한다
   - 무거운 작업은 백그라운드에서 처리해서 사용자는 기다리지 않는다

2. **스피킹 연습하기**
   - AI가 TTS로 한국어 문장을 읽어준다
   - 랜덤으로 문장을 선택해서 매번 다른 문제를 푼다
   - 내 목소리를 녹음해서 답한다

3. **음성 인식 & 채점**
   - Whisper AI가 내 음성을 텍스트로 변환한다
   - 정답과 비교해서 맞았는지 체크한다
   - 틀린 부분에 대한 피드백을 제공한다

## 🏗️ 기술 스택

### Frontend
- **Next.js 15** (App Router) - 최신 버전으로 SSR과 클라이언트 렌더링 둘 다 활용
- **React 19** - UI 컴포넌트 구성
- **TypeScript** - 타입 안정성 확보
- **Tailwind CSS** - 빠른 스타일링

### Backend
- **FastAPI** - Python 기반의 빠른 API 서버
- **PostgreSQL** - 문장 데이터, 사용자 정보, 학습 기록 저장
- **Redis** - 캐싱과 세션 관리에 사용

### Message Queue & Worker
- **RabbitMQ** - 작업을 큐에 넣어서 순서대로 처리
- **Celery** - 백그라운드에서 무거운 작업을 비동기로 처리
  - OCR/이미지 처리
  - TTS 파일 생성
  - Whisper 음성 인식

### AI/ML
- **OpenAI Whisper** - 음성을 텍스트로 변환
- **TTS API** - 텍스트를 음성으로 변환
- **Vision AI** - 이미지 속 문장 인식 (OCR)

### Infrastructure
- **Docker & Docker Compose** - 모든 서비스를 컨테이너로 관리
- **GitHub Actions** - CI/CD 파이프라인 (예정)

## 📊 데이터가 어떻게 흐르는가?

### 1. 문장 등록할 때 (이미지 업로드 케이스)
```
사용자가 이미지 업로드
    ↓
Frontend → Backend API (업로드 요청)
    ↓
Backend가 RabbitMQ에 "OCR 작업 해줘!" 메시지 전송
    ↓
Celery Worker가 큐에서 작업을 꺼내서 OCR 처리
    ↓
결과를 PostgreSQL에 저장
    ↓
사용자에게 "등록 완료!" 알림
```

**왜 이렇게 복잡하게?**
이미지에서 문장 추출하는 건 시간이 오래 걸린다. 사용자가 5초씩 기다리게 하면 답답하니까, 일단 "처리 중이야!"라고 빠르게 응답하고 백그라운드에서 천천히 처리하는 것이다.

### 2. 스피킹 연습할 때
```
사용자가 "연습 시작" 버튼 클릭
    ↓
Backend가 PostgreSQL에서 랜덤 문장 조회
    ↓
RabbitMQ → Celery Worker가 TTS로 음성 파일 생성
    ↓
Backend가 오디오를 사용자에게 전달
    ↓
사용자가 한국어 음성을 듣는다
```

### 3. 내 답변을 채점할 때
```
사용자가 음성 녹음
    ↓
Frontend → Backend API (오디오 파일 전송)
    ↓
RabbitMQ → Celery Worker가 Whisper로 음성→텍스트 변환
    ↓
정답 문장과 비교해서 평가
    ↓
사용자에게 "정답!" 또는 "틀렸어요, 이렇게 말해보세요" 피드백
```

## 🚀 왜 이런 기술들을 쓰는가?

### RabbitMQ + Celery를 쓰는 이유
처음엔 "그냥 FastAPI에서 다 처리하면 되지 않나?"라고 생각할 수 있다. 하지만:

- **OCR은 느리다**: 이미지 처리에 3~5초 걸린다
- **Whisper도 느리다**: 음성 인식에 5~10초 걸린다
- **TTS도 느리다**: 음성 파일 생성에 2~3초 걸린다

이런 작업들을 FastAPI에서 바로 처리하면 사용자는 계속 기다려야 한다. 그래서:
1. FastAPI는 빠르게 "알았어! 처리 중이야!" 응답
2. 무거운 작업은 RabbitMQ 큐에 넣는다
3. Celery Worker가 백그라운드에서 천천히 처리
4. 결과는 DB에 저장하고 나중에 조회

이렇게 하면 **사용자 경험이 좋아진다**. 그리고 Worker를 여러 개 띄워서 **병렬 처리**도 가능하다.

### Docker를 쓰는 이유
"내 컴퓨터에선 되는데요?"를 없애기 위해서다. Docker를 쓰면:
- 개발 환경과 프로덕션 환경이 똑같다
- PostgreSQL, RabbitMQ, Redis 등을 일일이 설치할 필요 없다
- `docker-compose up` 한 줄이면 모든 서비스가 뜬다

### Redis를 쓰는 이유
- **캐싱**: 자주 사용되는 문장 데이터를 메모리에 저장해서 빠르게 조회
- **세션 관리**: 사용자 로그인 상태를 저장
- **Celery 결과 저장**: 작업이 완료됐는지 빠르게 확인

## 🛠️ 개발 환경 세팅하기

### 필요한 것들
- **Docker Desktop** - 컨테이너 실행 환경
- **Node.js 20+** - Frontend 개발
- **Python 3.12+** - Backend 개발
- **uv** - Python 패키지 관리자 (pip보다 빠름)

### 실행 방법
```bash
# 모든 서비스 한 번에 시작 (PostgreSQL, RabbitMQ, Redis 등)
docker-compose up -d

# Frontend 개발 모드 (Hot Reload)
npm run dev:fe

# Backend 개발 모드 (Auto Reload)
npm run dev:be
```
